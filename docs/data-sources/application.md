# asgardeo_application (Data Source)

Use this data source to retrieve details about an existing Asgardeo application,
including its `client_id`. This is useful for referencing built-in applications
or applications created outside of Terraform.

## Example Usage

```terraform
# Look up an existing application by name.
data "asgardeo_application" "by_name" {
  name = "My Existing App"
}

# Look up an existing application by ID.
data "asgardeo_application" "by_id" {
  id = "e98e-4a6f-bd81-1234567890ab"
}

output "app_client_id_by_name" {
  value = data.asgardeo_application.by_name.client_id
}

output "app_client_id_by_id" {
  value = data.asgardeo_application.by_id.client_id
}
```

## Schema

### Optional

- `id` (String) Asgardeo application ID. Used as the lookup key when provided.
- `name` (String) Application display name. Used as the lookup key when `id` is not provided.

### Read-Only

- `access_url` (String) URL users are directed to when launching the application.
- `application_enabled` (Boolean) Whether the application is currently enabled.
- `client_id` (String) OAuth2 / OIDC client ID (if an OIDC protocol is configured).
- `description` (String) Description of the application.
- `inbound_protocol_types` (List of String) List of inbound protocol types configured for the application (e.g. `oidc`, `saml`).
- `logout_return_url` (String) Post-global-logout redirect URL.
