# Asgardeo Provider

The **Asgardeo** Terraform provider enables infrastructure-as-code management of
[Asgardeo](https://wso2.com/asgardeo/) — WSO2's cloud-native customer identity
and access management (CIAM) platform.

Use the provider to:

- Register OIDC and SAML applications
- Manage OAuth2 grant types, callback URLs, and allowed origins
- Configure advanced settings (consent, token expiry, PKCE)
- Look up existing applications via the data source

## Authentication

Create an **M2M application** in the Asgardeo Console, then authorise it with
the management API scopes below:

| Scope | Used for |
|---|---|
| `internal_application_mgt_create` | Creating applications |
| `internal_application_mgt_view` | Reading applications |
| `internal_application_mgt_update` | Updating applications |
| `internal_application_mgt_delete` | Deleting applications |

Credentials can be supplied via provider arguments or environment variables:

| Argument | Environment variable |
|---|---|
| `org_name` | `ASGARDEO_ORG_NAME` |
| `client_id` | `ASGARDEO_CLIENT_ID` |
| `client_secret` | `ASGARDEO_CLIENT_SECRET` |

## Example Usage

```terraform
terraform {
  required_providers {
    asgardeo = {
      source  = "asgardeo/asgardeo"
      version = "~> 0.1"
    }
  }
}

# Credentials can be passed as arguments or via environment variables:
#   ASGARDEO_ORG_NAME, ASGARDEO_CLIENT_ID, ASGARDEO_CLIENT_SECRET
provider "asgardeo" {
  org_name      = var.asgardeo_org_name
  client_id     = var.asgardeo_client_id
  client_secret = var.asgardeo_client_secret
}
```

### Optional

- `client_id` (String) Client ID of the M2M application used to authenticate against the Asgardeo Management API. Can also be set via `ASGARDEO_CLIENT_ID`.
- `client_secret` (String, Sensitive) Client secret of the M2M application. Can also be set via `ASGARDEO_CLIENT_SECRET`.
- `org_name` (String) Asgardeo organisation name (the subdomain part of your Asgardeo URL). Can also be set via the `ASGARDEO_ORG_NAME` environment variable.
